# Web Frontend

The `web/` directory contains a single-page application that consumes the egras REST API. It lives in the same repository as the backend so the OpenAPI spec is the single source of truth and never drifts.

## Stack

| Concern | Choice |
|---------|--------|
| Bundler / dev server | [Vite 5](https://vitejs.dev/) |
| Framework | React 18 + TypeScript |
| Styling | Tailwind CSS v3 |
| Routing | React Router v6 |
| Server state | TanStack Query v5 |
| API client | [`openapi-fetch`](https://openapi-ts.dev/openapi-fetch/) with types generated by `openapi-typescript` |

## How it stays in sync with the backend

`docs/openapi.json` (committed, drift-checked in CI) is the source. The `gen:api` npm script runs `openapi-typescript ../docs/openapi.json -o src/api/schema.d.ts` and is invoked automatically by `npm run dev` and `npm run build`. There is no separate publish step or version pin.

Generated `schema.d.ts` is gitignored ‚Äî regenerated on every dev/build.

## Auth

JWT is stored in `sessionStorage` under `egras.jwt`. The `openapi-fetch` middleware in [`web/src/api/client.ts`](../../web/src/api/client.ts) injects `Authorization: Bearer ‚Ä¶` on every request and redirects to `/login` on any `401`.

## Running locally

```bash
# Terminal 1: backend
EGRAS_DATABASE_URL=postgres://postgres:secret@localhost:5432/egras \
EGRAS_CORS_ALLOWED_ORIGINS="http://localhost:3000" \
  cargo run

# Terminal 2: frontend
cd web
npm install
npm run dev   # http://localhost:3000
```

Vite proxies `/api/*` to the backend on `:8080`, so the browser sees a same-origin app.

## Pages

| Route | Backend endpoint(s) | Purpose |
|-------|---------------------|---------|
| `/login` | `POST /security/login` | Sign in |
| `/register` | `POST /security/register` | Self-register into an org |
| `/reset-password` | `POST /security/password-reset-{request,confirm}` | Two-stage reset |
| `/organisations` | `GET /tenants/me/organisations`, `POST /tenants/organisations` | Caller's orgs + create |
| `/organisations/:id/members` | `GET /tenants/organisations/{id}/members`, `POST /tenants/{add,remove}-user-to-organisation` | Membership management |
| `/organisations/:id/channels` | `* /tenants/organisations/{org_id}/channels[/{id}]` | Inbound channel CRUD |
| `/users` | `GET /users` | Cross-tenant user list (operator only) |
